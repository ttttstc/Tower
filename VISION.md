# Tower — 产品愿景

> 像看一座塔一样看你的项目。

---

## 一句话定义

Tower 是一个**视觉系项目管理工具**，把项目的事项层级转换为一座应县木塔：每个事项是一块方块，颜色代表状态。项目管理者打开工具的第一眼，就能像看一座古塔一样看到整个项目的健康度——哪里在砌、哪里阻塞、哪里地基已经封顶。

---

## 为什么做这个

所有现存 PM 工具——Jira、Linear、Notion、飞书——都在拼功能广度或协同深度。它们的主视图永远是列表、看板、甘特图：抽象、扁平、需要认知解码。

**没有任何主流工具用一个具象的、美的、一眼可读的建筑物作为唯一主视图。**

Tower 占据这个空白。它不试图取代 Jira，而是给在乎视觉的小团队、独立产品人、技术 Lead 一个赏心悦目的项目仪表盘。差异化不在功能，在**美学锐度和一眼可读性**。

---

## 核心隐喻

### 应县木塔

灵感来源是山西应县佛宫寺释迦塔——中国现存最古老的木结构塔。五层明层，每层飞檐出挑，底宽顶收，比例庄严。

Tower 的主视图是一座数字化的应县木塔：

- **五大楼段** = 项目的五个事项层级
- **楼段内部** = 股票大盘式的 Treemap 云图，方块紧密填充
- **方块面积** = 工作量（effort）
- **方块颜色** = 事项状态
- **层间飞檐** = 装饰性分隔，飞檐中央标注楼段名称
- **塔刹** = 塔顶标志，代表项目愿景的最终达成

### 五大楼段（自上而下）

| 楼段 | 含义 | 典型事项 |
|------|------|----------|
| 愿景层 | 项目目标与验收标准 | 愿景对齐、里程碑规划 |
| 设计层 | 架构方案与技术设计 | 架构设计、方案评审 |
| 协同层 | 跨团队/跨模块协调 | 接口联调、权限协同 |
| 实现层 | 具体开发任务 | 编码、单测、集成 |
| 基础设施层 | 环境/工具链/底层依赖 | 部署、监控、镜像 |

顺序固定不变。塔从基础设施层（底部）向上生长到愿景层（顶部），符合建筑和项目的双重逻辑。

## 状态色彩

五种状态，五种颜色，无歧义：

| 状态 | 颜色 | 含义 |
|------|------|------|
| 完成 | 绿色 | 事项已闭环 |
| 进行中 | 暖色（琥珀/砖橙） | 正在推进 |
| 待启动 | 灰色 | 已录入但未开始 |
| 延期 | 黄色 | 超过预期时间 |
| 阻塞/风险 | 红色 | 被阻塞或存在风险 |

单个方块完成 → 变绿。一个楼段内所有方块全部完成 → 整个楼段背景变为绿色（封顶效果）。

## 核心交互

### 主视图

- 首页默认按比例缩小，展示完整的塔身全貌
- 支持滚轮缩放和屏幕上的 +/- 按钮，可放大查看某一楼段的内部方块
- 楼段内部为 Treemap 云图，方块紧密填充，面积正比于工作量
- 方块内只显示事项标题；面积不够时截断或隐藏文字

### Hover 挤压

- hover 某个方块时，该方块平滑扩张，同层内其他方块被挤压收缩
- 扩张不改变 z-index，不悬浮覆盖，而是在同层内推开周围方块
- 距离越近的方块被挤压越少，距离越远被压缩越多
- 楼层容器总面积保持不变
- 所有动效由弹簧物理（Spring Physics）驱动，黏糊且顺滑

### 依赖线

- 默认所有依赖线不可见
- hover 方块时，该方块的所有上下游跨楼段依赖线在 150ms 内淡入
- 鼠标移开 200ms 后线条淡出
- 线条样式：1px 暖色线，从下方砖顶部到上方砖底部

### 责任人面板

- 页面右侧常驻责任人清单，默认全选
- 支持全选、全不选、单个/多个勾选筛选
- 选中某些人名时，对应方块高亮，其余方块暗淡
- 责任人面板是主页面的一部分，不是抽屉

### 详情抽屉

- 点击任意方块，详情抽屉从右侧滑入，覆盖在责任人面板之上
- 抽屉内可编辑事项的所有字段：标题、楼段、依赖、工作量、负责人、状态、风险标记、备注
- 编辑保存后方块视觉立即更新

---

## 数据与持久化

- 本地优先，所有数据存储在单一 `tower.json` 文件中
- 通过 File System Access API 读写（仅 Chromium 系浏览器）
- 提供 JSON Schema，用户可将 tower.json 纳入 Git 管理
- 每日首次打开时自动快照到 `snapshots/YYYY-MM-DD.json`

---

## 日报

- 点击"今日日报"按钮，全屏抽屉显示
- 对比昨日快照与当前状态
- 内容：新增的方块、状态变化的方块、新封顶的楼段
- 纯文本 Markdown 格式，支持复制到剪贴板

---

## 目标用户

在乎视觉的小团队（5-15 人）、独立产品人、技术 Lead。典型场景：每天早上打开 Tower，10 秒内扫一眼塔的状态分布，定位今天要推的阻塞点。

---

## MVP 不做

- Excel 导入（v1.1）
- IM 通知 / Webhook（v1.2）
- 云端后端 / 多人实时协作
- 移动端 / 触屏适配
- 暗色主题（v1.1）
- Safari / Firefox 支持
- 国际化（仅中文 UI）
- 甘特图 / 时间轴视图
- 传统看板视图

---

## 成功标准

打开一个含 ≥20 个事项的项目，旁观者在 5 秒内能说出："这是一座塔，每块砖是一个事项。"
